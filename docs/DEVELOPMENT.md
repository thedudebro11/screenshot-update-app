# ScreenMonitor — Development Guide

## Prerequisites

- **Node.js** 18+ and **npm**
- **Windows** (required — `desktopCapturer` and the tray icon work on Windows; macOS needs `screen capture` permission; Linux needs additional setup)
- **Git**

---

## First-Time Setup

```bash
git clone https://github.com/thedudebro11/screenshot-update-app
cd screenshot-update-app
npm install
```

Download `cloudflared.exe` for local dev (required for the tunnel):
```bash
node scripts/download-cloudflared.js
```

This places `bin/cloudflared.exe` next to the source files. The download is skipped if the file already exists.

---

## Running in Development

```bash
npm start          # normal dev run
npm run dev        # with --enable-logging (Electron internal logs visible in terminal)
```

The app starts in the system tray. Right-click the green icon to interact with it.

The local viewer URL is:
```
http://localhost:3456/?token=screenmonitor
```

**Dev vs. production differences:**
| Aspect | Dev (`npm start`) | Production (installed .exe) |
|--------|-------------------|------------------------------|
| Data directory | project root | `%APPDATA%\ScreenMonitor\` |
| `.env` location | project root | `%APPDATA%\ScreenMonitor\` |
| Screenshots | `./screenshots/` | `%APPDATA%\ScreenMonitor\screenshots\` |
| Logs | `./logs/app.log` | `%APPDATA%\ScreenMonitor\logs\app.log` |
| cloudflared | `./bin/cloudflared.exe` | `resources/cloudflared.exe` |
| Auto-update | Disabled (not packaged) | Enabled |
| Auto-startup | Enabled (but points to dev electron) | Enabled (points to installed exe) |

## Building the Installer

```bash
npm run dist
```

This runs `predist` first (automatically), which:
1. `node scripts/generate-icon.js` — writes `build/icon.ico` (16, 32, 48, 256px green PNG embedded in ICO format, no external deps)
2. `node scripts/download-cloudflared.js` — downloads `bin/cloudflared.exe` from GitHub if not present

Then `electron-builder --win` produces:
```
dist/
├── ScreenMonitor Setup 1.0.0.exe    ← the installer
└── ...                              ← unpacked build artifacts
```

**The installer:**
- Does NOT require Node.js on the target machine
- NSIS installer (not one-click — user can choose install directory)
- Installs to `C:\Program Files\ScreenMonitor\` by default
- Starts the app immediately after installation

## Publishing an Update

1. Bump `version` in `package.json`
2. Commit and push
3. Create a GitHub tag: `git tag v1.0.1 && git push origin v1.0.1`
4. Run `npm run dist` — produces the new installer
5. Create a GitHub Release at `github.com/thedudebro11/screenshot-update-app/releases`
6. Attach the `dist/ScreenMonitor Setup x.y.z.exe` to the release

Running instances will detect the update on their next startup and display a balloon: "A new version has been downloaded. Restart the app to apply it."

The auto-updater uses `electron-updater` pointed at the GitHub provider config in `package.json`:
```json
"publish": [{ "provider": "github", "owner": "thedudebro11", "repo": "screenshot-update-app" }]
```

## Modifying the Web Viewer

`web/index.html` is a single self-contained file — no bundler, no npm packages, no build step. Just edit and reload.

**In dev:** Hard-refresh the browser (Ctrl+Shift+R) — the file is served directly by Express.

**The CSS design system** uses custom properties defined in `:root`:
```css
--bg:     #09090b  /* page background */
--accent: #22c55e  /* green — LED, active dots, active buttons */
--warn:   #f59e0b  /* amber — fallback status */
--error:  #ef4444  /* red — error status */
--font:   'JetBrains Mono', ... monospace
```

**Adding a new API route:**
1. Add the route handler in `server.js`
2. Add auth middleware will apply automatically (it runs on `app.use()`)
3. Call it from the web viewer with `api('/your-route')` (appends the token automatically)

**The `api()` helper:**
```js
const api = (p) => p + (p.includes('?') ? '&' : '?') + `token=${TOKEN}`;
// api('/status')          → '/status?token=screenmonitor'
// api('/screenshot?t=1')  → '/screenshot?t=1&token=screenmonitor'
```

## Key Variables in `web/index.html`

```js
let screenshots = [];   // [{ t, iso }, ...] newest first — source of truth for timeline
let currentT    = null; // selected screenshot timestamp, null = latest
let isLive      = true; // false when browsing history
let isOnline    = true; // false when server unreachable
let countdownN  = 30;   // seconds until next auto-refresh
let tickId      = null; // setInterval ID for the countdown

const z = { scale: 1, dx: 0, dy: 0 }; // zoom state
```

**Important behavioral rules:**
- `selectShot(t)` sets `isLive = false` but does NOT clear the countdown interval
- `updateCountdown()` is a no-op when `isLive = false` — the timer runs silently
- `goLive(resetTimer=true)` fully resets the countdown; `goLive(false)` does not
- The Refresh button calls `goLive(false)` — it loads the latest and goes live without restarting the timer
- The Live button calls `goLive()` (default true) — resets the countdown

## Debugging Tips

### App won't show in tray
- Check if another instance is running (Task Manager)
- Run `npm run dev` to see Electron error output in terminal

### Screenshots are black
- The target window uses hardware acceleration (games, Chrome with GPU acceleration)
- No workaround — it's a Windows/Electron limitation

### Target window not found
1. Open `logs/app.log`
2. Find `Available windows:` lines — shows pipe-separated window titles
3. Copy the partial title into `TARGET_WINDOW_TITLE` in `.env`
4. Restart the app

### Tunnel not connecting
1. Check `logs/app.log` for `[tunnel]` lines
2. Verify `bin/cloudflared.exe` (dev) or `resources/cloudflared.exe` (packaged) exists
3. Check Windows firewall isn't blocking outbound connections on port 7844 (Cloudflare's tunnel port)
4. The tunnel URL appears in the log as `Tunnel live: https://...`

### Web viewer not updating
1. Open browser DevTools → Network → filter by "events" to see the SSE stream
2. Heartbeat frames every 25s means the SSE connection is healthy
3. If SSE isn't connecting, check the 30s poll fallback (visible in the countdown)
4. Check `logs/app.log` for "No change detected" — the app IS running but the screen is static

### Build fails
- Run `predist` scripts manually to isolate the issue:
  ```bash
  node scripts/generate-icon.js
  node scripts/download-cloudflared.js
  ```
- Ensure `build/icon.ico` exists before running `electron-builder`
- Ensure `bin/cloudflared.exe` exists before running `electron-builder`

### Auto-update not working
- Only runs when `app.isPackaged` is true (installed exe, not `npm start`)
- Requires a GitHub Release with the correct version tag matching `package.json`
- Check `logs/app.log` for `electron-updater` output

## Adding a New Capture Action

The capture loop runs `doCapture()` in `main.js`. The result from `captureWindow()` in `capture.js` is typed:

```js
// success path
{ success: true, pngBuffer: Buffer, windowName: string, isFallback: boolean }

// error path
{ success: false, error: string, availableWindows: string }
```

Never throw from `captureWindow()` — return an error result object instead. The capture loop is the main hot path of the app and must be robust.
